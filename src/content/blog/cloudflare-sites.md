# 我部署过的几个 Cloudflare 网站

这几个网站对我来说不是单纯的域名列表，更像是我开始真正把项目部署到互联网上的几个节点。

它们的时间顺序大概是：

1. 先从 GitHub 找资源，参考教程部署项目；
2. 再部署 NewsNow 这种现成的信息聚合项目；
3. 最后第一次用 Codex 帮我写了一个自己的网页。

## 1. GitHub 资源部署记录

入口：[lsldsb.dpdns.org](https://lsldsb.dpdns.org)

这是我第一次正式在 GitHub 上找资源、参考教程部署出来的项目。

这个项目是我很喜欢的一位大佬“零度解说”推荐过的，属于影视、直播、资源类项目。它对我的意义不只是“能打开一个网站”，而是让我第一次完整走了一遍：

- 找到一个 GitHub 项目；
- 看教程理解大概怎么跑起来；
- 在自己的电脑上试着部署；
- 再把它放到 Cloudflare 上，让它变成一个可以公开访问的网站。

这个过程让我意识到，很多看起来很远的东西，其实可以一步一步拆开做。

## 2. NewsNow 信息聚合站

入口：[newsnow-76c.pages.dev](https://newsnow-76c.pages.dev/)

NewsNow 是 GitHub 上的一个信息聚合类项目，可以用来看各大主流平台的信息。

我当时把它部署到了 Cloudflare Pages。最开始还给它用过 `140035.xyz` 这个域名。后来我用 Codex 写了另一个自己的网站，就把 `140035.xyz` 换给了那个新网站。

这一步让我对“项目”和“域名”的关系有了更直观的理解：域名只是入口，它可以指向不同的网站；真正重要的是背后的项目是什么、解决了什么问题。

## 3. 140035.xyz 手机卡推荐页

入口：[140035.xyz](https://140035.xyz/?from=li1408-blog)

这是我第一次接触 Codex 时做出来的网站。

当时我的想法很简单：做一个大流量手机卡推荐页面，风格参考苹果官网那种大留白、圆润、干净的设计。我没有从头手写完整页面，而是用很简单的语言告诉 Codex 我想要什么，再让它帮我生成和调整。

它不一定是技术上最复杂的项目，但它对我很重要。因为这是我第一次感觉到：只要能把想法说清楚，AI 工具真的可以帮我把一个网页从想法变成能访问的东西。

## 小结

这三个网站刚好代表了三个阶段：

- `lsldsb.dpdns.org`：第一次跟着 GitHub 资源和教程部署项目；
- `newsnow-76c.pages.dev`：第一次把信息聚合类项目放到 Cloudflare Pages；
- `140035.xyz`：第一次用 Codex 做出更像“自己的网页”的东西。

它们不是终点，更像是我开始折腾网站、域名和部署的起点。
