see140 最近做了一次比较大的更新。

在线地址是：[li1408.github.io/see140](https://li1408.github.io/see140/)

GitHub 仓库是：[github.com/li1408/see140](https://github.com/li1408/see140)

这次更新不是简单加几个按钮，而是继续打磨它作为一个网页互动作品的完整度。原来的 see140 已经可以通过电脑摄像头识别手势，在网页里进行 3D 线条绘制、重力下落、双手缩放和手势清除。

但我后来发现，一个作品“能用”和“真的有体验感”中间还有一段距离。

这次更新主要围绕两个方向：

- 手势控制更完整。
- 视觉反馈更明显。

我希望用户不是只看到线条被画出来，而是能明显感觉到：自己的手势正在影响这个画面。

## 加入新的手势控制

这次加入了新的手势控制。

现在可以用 **单手 Victory 手势** 长按约 1 秒切换 Mode，也就是在普通画板模式和重力模式之间切换。

这个设计解决了一个很实际的问题：以前模式切换更依赖 UI 操作，但 see140 本身是一个摄像头手势交互作品，如果核心模式还要回到鼠标或按钮上切换，就会有一点割裂。

所以这次把模式切换也放进了手势系统里。

另外，我还加入了 **双中指** 手势，用来切换 Wind 风力状态。

这个手势看起来有点突兀，但我反而觉得它适合 see140。因为这个项目本来就不是一个非常正经的办公工具，它更像一个带有实验感的视觉交互作品。手势本身有一点怪，反而会让人记住。

为了避免误触，Victory 和双中指都不是一出现就立即触发，而是需要保持一小段时间。这样可以让操作更稳定，也避免识别过程中的偶然姿势把模式切走。

## 风力只在重力模式里生效

这次风力系统也重新打磨了一遍。

我没有让风在所有模式里都生效，而是让它只在 **重力模式** 下生效。

原因很简单：普通画板模式里，线条应该更像一张安静的画布，用户画出来的东西会停留在原地。如果风在这个模式里也不断影响线条，画板就会变得不稳定。

但重力模式不一样。

重力模式本来就强调线条的物理感：下落、碰撞、回弹。风力加入这里，逻辑上更自然。线条不再只是往下掉，而是会受到手掌位置和风力强度影响，产生更像“空间中物体”的运动。

## 让风真正被看见

之前风的存在感不够明显。

有时候代码里确实有风力计算，但用户看不太出来。对一个互动作品来说，这其实就是失败的反馈。用户不应该猜测“是不是有风”，而应该一眼感觉到“风正在推动画面”。

这次我把风改成更稳定、更可感知的交互。

风不再只依赖快速挥手，而是会根据手掌在画面中的位置生成方向感：

- 手在左边，线条会被往右推。
- 手在右边，线条会被往左推。

同时 UI 中加入了风力强度滑块，可以直接调整风的强弱。

这个滑块很重要。因为风力太弱，用户看不出来；风力太强，画面又会失控。把强度交给用户调节，比写死一个数值更合理。

## 粒子反馈变得更明显

这次清除线条的粒子效果也被强化了。

清除不再只是“线条消失”，而是线条会变成更多、更明显的粒子。粒子不会立刻淡掉，而是先保持一小段高亮，再扩散、淡出。

我很喜欢这个变化。

因为清除本来只是一个功能，但在 see140 里，它应该成为一个动作反馈。用户做出清除手势后，画面应该给出足够明确的回应，让人感觉到这个动作真的发生了。

吹风时，空气中的细微粒子也被增强了。现在粒子数量更多，尺寸更明显，并且会随着风力产生流动和扰动。

这样一来，风就不只是一个看不见的物理参数，而是变成了画面里可以被感知的东西。

## 补上测试

这次也补了一些测试。

主要覆盖了几个核心逻辑：

- 手指识别。
- 手势保持时间。
- 风力方向和强度计算。

我以前更关注“效果能不能跑起来”，但越往后越发现，手势交互这种东西很容易在修改 UI 或动画时被不小心改坏。

测试的意义不是让项目看起来更正规，而是给后续继续打磨留一点底。这样以后再改手势、风力、粒子效果时，不至于每次都从头怀疑基础逻辑是不是坏了。

## 技术实现简述

see140 主要使用 **Next.js、Three.js 和 MediaPipe** 实现。

MediaPipe 负责从摄像头画面中识别手部关键点，Three.js 负责渲染 3D 线条、重力效果、风力运动和粒子动画。

这次更新里，我额外补了手指状态判断逻辑，用来识别 Victory 和双中指手势。

风力部分则通过手掌位置计算方向和强度。它不是单纯判断用户有没有挥手，而是把手掌在画面里的位置转成一个持续的力，让线条和粒子产生更稳定的运动反馈。

清除和吹风的视觉反馈，主要通过 Three.js 的粒子系统实现。清除时把线条拆成粒子，吹风时让环境粒子跟随风力方向流动。

## 项目来源说明

see140 前期参考和借鉴了 Cygra 的开源项目 Hand Gesture Whiteboard：

[github.com/Cygra/hand-gesture-whiteboard](https://github.com/Cygra/hand-gesture-whiteboard)

原项目提供了基于 Next.js、MediaPipe Gesture Recognizer 和 Three.js 的手势白板基础能力。

在这个方向上，我逐步加入了自己的交互设计、3D 线条效果、重力物理、风力系统、粒子清除、启动动画和黑白 UI 风格，最后形成了现在的 see140。

## 这次更新给我的感受

这次更新让我有一个很明显的感受：很多时候平台比能力重要。

这里的“平台”，不只是 GitHub Pages 这种部署平台，也包括 GitHub、Codex、开源项目、浏览器、摄像头、网页技术这些东西共同组成的环境。

如果没有这些平台和工具，一个像 see140 这样的想法，可能只会停在脑子里。

但当这些平台都摆在面前时，事情就变了。一个模糊的想法可以被拆成需求，一个粗糙的 demo 可以被不断修改，一个网页可以被部署出去，一个别人开源的方向也可以成为自己继续创作的起点。

能力当然重要。

但很多时候，人真正缺的不是一开始就很强的能力，而是一个能让自己不断试错、不断看到反馈、不断把想法变成东西的平台。

see140 这次更新对我来说，不只是功能升级。它更像是在提醒我：如果一个人能找到合适的平台，并且愿意在上面持续折腾，很多原本看起来离自己很远的东西，其实也可以一点点做出来。
